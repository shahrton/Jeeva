Overview

This section applies to all platforms. (when you see text this color, hover your mouse over it.)

This is a file/folder utility callable from the command line, e.g.: jeeva [options] folder1 folder2

When called with option

--overlap
(was --compare) Compares 2 folders or folder trees. Returns the number (and optionally the names) of files that are in one or the other folder, or common to both. A read-only operation, no copying is done. Can recursively descend (optionally to a specifed depth) down the folder tree. Does not yet compare the actual file contents or look at modification dates, but these features will be added late 2019/early 2020.
--mtime
Compares 2 folders or folder trees. Returns the number (and optionally the names) of files that are newer, the same age, or older in folder1 vs folder2. Can recursively descend (optionally to a specifed depth) down the folder tree.
--copy
used for mirroring folders, primarily useful for keeping a mirror on another hard drive or for frequent copying to a USB stick. Has a short copying time as allows one to selectively copy over to the destination folder only those files that are newer in source than in the destination folder; useful when the total number of files is large but the number of newer files is relatively small.

For any of the above options one can either:

  • specify the Source and Destination folders or
  • specify 2 folders that we refer to as the roots of the Source and Destination folder systems, and then run Jeeva from a folder somewhere below the source root. In this case Jeeva will take that folder to be the source, will look for a folder of the same name and in a corresponding position below the destination root, and define that to be the Destination folder. It shall then proceed as usual to copy or compare.

Requirements

Python
  • Jeeva is written in Python. However MS Windows users do not necessarily have to install Python.
  • Mac OSX, Linux and Unix users: your computer almost certainly has Python (if your OS is newer than Mac OSX 10.3) but if you want to verify: Open a command line window, Terminal or iTerm, (called a console or Xterm for Linux/Unix users). Type python --version. As long as the version is later than 2.5 you should have no problem. If you see "3.x.x" you have Python but unfortunately it is Python 3, and the software is not written for Python 3 yet, expect around Feb 2020.
  • MS Windows users: You might already have Python, or you might wish to install it, or you could try using the the pre-built Jeeva binary/executable provided (created using the PyInstaller software utility), for which you will not need to install Python on your computer.

Installing Jeeva

Linux, Unix, MAC OSX and MS Windows+Python users use similar installation instructions since they use the Python interpreter already on their computer to interpret the source code. MS Windows no-Python instructions are different since pre-compiled executable is provided.

MS Windows no-Python users who so desire can look at the source code (link provided at top of ths doc). This has extra CR/LFs inserted to make viewing in Windows Notepad easier. Other users get source code anyway.

Installing in Linux, Unix or Mac OS X

Download either the zip or the tar file, which you will find after clicking on the Files tab on that page. unzip or un-tar the downloaded file into its own folder, which you will find to contain several files.

nb: Experienced users can skip following.

You can set up a shortcut icon on your Mac/Linux desktop to point to the location of the file jeeva.py. But if you want to start from a terminal/console the following sets up to run whenever you type Jeeva at the command line, with one of the 2 following methods:
Using an alias (easier method)
Put an alias into your .login file as follows:
	  alias Jeeva "python path_to_Jeeva_folder/jeeva.py"
You might have to re-login, or open a new Terminal, or type source .login for the change to take effect. You might prefer to put the alias command into the files .cshrc, .bashrc or .kshrc but I think .login is the best choice.
Using a soft link
  • Make a soft link to jeeva.py from a folder already in your command path. echo $PATH will show you folders in your command path. Go to one such folder & make a soft link to Jeeva.py: 
    		  ln -s Path_to_Jeeva_folder/jeeva.py    Jeeva
    Many people keep a specific folder, such as $HOME/bin, to hold soft links to point to frequently used programs and utilities. But any folder in the command path will work, even the folder you just extracted your download to provided you put that folder in the command path.
  • Typing rehash followed by which Jeeva should give the pathname to the soft link you just made.
  • Ensure that the first line in the file jeeva.py looks something like: #!/usr/local/bin/python accurately reflects where your python interpreter executable is. If in doubt, type which python, and make that first line in jeeva.py reflect the location of the Python interpreter, taking care to enter the "#!" characters.
  • The execute permission on the file jeeva.py should already be set correctly. But if you get a Permission denied error message later when trying to run Jeeva, then type 
    chmod u+x Path_to_Jeeva_folder/jeeva.py
    and that should set your execute permission correctly and fix the problem.

Installing in MS Windows

Download & Install: Click here and download the zip file jeeva_1.6.2_Windows.zip which you will find after clicking on the Files tab on that page. The zip file contains a pre-compiled executable. Extract the contents of the zip. A directory/folder jeeva will contain jeeva.exe and several other files which are important as they contain and use a Python interpreter so do not delete them. Either add this folder to your PATH environment variable, or choose a folder already in your PATH, and add a file named jeeva.bat which will consist of something like the following lines: 

  @echo off 
  REM This is to run jeeva, a folder comparison and mirroring script
  C:\Users\YOURUSERNAME\PLACE_YOU_DOWNLOADED_TO\Jeeva\jeeva.exe %*
  REM or an example with with Source and Destination roots hardwired:
  C:\Users\YOURUSERNAME\PLACE_YOU_DOWNLOADED_TO\Jeeva\jeeva.exe --sroot C:\Users\YOURUSERNAME --droot F:\Destination_Folder %*
i.e. point to the location to which you extracted the zip file contents. (There is also a file sample_jeeva.bat in the unzipped folder.) Hopefully then, typing jeeva at a DOS prompt will work: Open a DOS window using cmd and test by typing jeeva --version or jeeva --help.

I test Jeeva on a Windows 7 computer and am unsure whether it will work on newer Windows platforms. If jeeva --version works, that is a good sign. If not, it might be necessary to experiment with the Compatibility: go to the jeeva.exe file ==> right-click ==> Properties ==> Compatibility

Upgrading Jeeva

Simply uninstall/delete your previous distribution and replace it with the new one. Or install the new one in a new location and edit your jeeva.bat file to point to it.

Using Jeeva

The --overlap, --mtime and --copy options described later can be run in one of 2 modes, either by specifiying Source and Destination folders: 

jeeva --overlap/--mtime/--copy   Source_folder    Destination_folder
or 2 folders that you call roots of the Source and Destination folder systems are specified: 
jeeva --overlap/--mtime/--copy   --sroot root_Source_folder    --droot root_Destination_folder
The first mode is more useful for running Jeeva once for a full directory tree.

The second mode Jeeva is run from root_Source_folder or a folder below it, and that folder is taken to be the source folder. Jeeva will look for a folder of the same name and in a corresponding position below root_Destination_folder, and will define that to be the Destination folder. It will then proceed as it would for the first mode to copy or compare. This mode of operation is more useful than the first as it allows you to wander around in the source root folder tree and perform multiple compares and copys using the same repeated Jeeva command every time. Further, if your source and and destination roots are always the same you can hardwire them into your jeeva.bat file, (Apple/Linux: into a shell script or alias command) and not bother with typing the whole thing every time.

The descriptions below are tailored to the first mode, but apply to the second mode too, with the understanding of what the Source and Destination folders are. If you are unclear about how this works, experiment with --overlap, or --copy with --dryrun.

Comparing overlap:
jeeva --overlap Source Dest compares the two directories Source and Dest, or, with --recursive, descends their directory trees and compares any corresponding directories. (--overlap used to be --compare.) For each directory 6 quantities are returned:
  1. Number of regular files in source directory but not destination directory
  2. Number of regular files in both source and destination
  3. Number in destination but not source,
  4. (and 5,6) The same 3 corresponding values but for directories instead of for regular files

Example: 

jeeva --overlap .\Videos    D:\Shaheen\Mirror\Videos
produces: 
Videos (23,0,0 ; 16,3,0)
indicating that the first (source) directory contains 23 regular files not in the second (destination) directory, 16 sub-dirs not in destination directory, and 3 sub-dirs are common to both directories. Adding --recursive produces similar output for the 3 sub-dirs that the source and destination have in common, going down 2 further sub-directory levels: 
Videos (23,0,0 ; 16,3,0)
   |->2017 (0,13,0 ; 0,0,0)
   |->Lumix (0,1,0 ; 1,2,0)
      |->Ohio 2018 (20,60,91 ; 0,0,0)
      |->India Trip Feb Mar 2018 (0,98,0 ; 0,0,0)
   |->2018 (2,7,0 ; 0,0,0)

With --depth 1 the display would only have gone down one level instead of all (both) levels. If --depth (or -d) is used, it is no longer necessary to also specify --recursive / -r.

With --details d_option option it displays actual file names in these 6 categories. d_option must be one of yes, query or a 6 character yes/no mask (e.g. "yyyyyy" or "ynynyn" or "nnnnnn") to select which of the 6 quantities you wish detailed.

If the directory tree is large the output could be voluminous. If you are only interested in seeing where the differences are between folders, add --skipsame which suppresses output for folders where source and dest have all files in common. If you are only interested in seeing where there are no differences between folders, add --onlysame which suppresses output for folders where source and dest do not have all file names in common.

Comparing file modification times:
jeeva --mtime Source Dest compares the two directories Source and Dest, or, with --recursive, descends their directory trees and compares any corresponding directories. For each directory 3 quantities are returned:
  1. Number of regular files newer in source directory than in destination directory
  2. Number of regular files same age in source directory and destination directory
  3. Number of regular files older in source directory than in destination directory

With --details d_option option it displays actual file names in these 3 categories. d_option must be one of yes, query or a 3 character yes/no mask (e.g. "yyy" or "yny") to select which of the 3 quantities you wish detailed.

--dt permits defining a time interval such that if the modifcation times of 2 files differ by less than it, then they are consdered the same age. Takes 2 string args: (1) Value itself and (2) formatter in same style used by datetime.strptime.
eg --dt \"1 2 3\", \"%H %M %s\" ==> file mtimes that differ by less than 1 hour 2 min 3 seconds are considered same age. Default value = 0. Cannot exceed 12 hours.

The other options --depth, --skipsame and --onlysame work much the same way as they do for the --overlap. e.g. --onlysame will only display stats where all the files in Source and Dest are the same age.

Copying:

jeeva --copy mod Source Dest : copies regular files that have more recent modification time in Source. Files that are in Source but not in Dest are not copied. With --recursive, does same while descending tree structure. Never copies directory files.

jeeva --copy new Source Dest copies regular files that are in Source but not in Dest. With --recursive, does same while descending tree structure. With --recursive --mkdir yes (or --mkdir query), additionally, directories that are in Source but not in Dest are created and copied along with their contents. If you answer yes to a query to copy a folder, (since shutil.copytree is used,) all its sub-folders will also be copied and you wil not have to laboriously descend its whole tree answering yes/no.

--copy newmod combines the 2 previous actions. Used to be --copy update

Option --dryrun shows what action would be taken without doing any actual copying, to allow a check beforehand.

Both Source and Dest must exist beforehand. I decided it is better that use your OS's directory tree copying utility to initially create Dest, rather than that I implement something like a jeeva --copy all Source Dest option. If from command line, MS Windows use xcopy or robocopy. Linux/Unix/Apple use cp -ar or cp -r

--ignorefile file1,file2,...: If present then the regular file names listed will not be copied. Useful especially in MS Windows which has an irritating habit of creating system files like desktop.ini and thumbs.db in every directory, and Jeeva attempts to copy them over to Destination, unsuccessfully because they are owned by System. If multiple files, separate with comma, no space. If Spaces present within a filename, surround whole option argument with double-quotes. Gotcha!: When directories are copied, they are done in entirety (with shutil.copytree) and no check is made whether files in that directory are on the --ignorefile list.

--ignoredir dir1,dir2,.... Similar to above item but also the files and sub-folders in these ignored directories will also be ignored.

Case-sensitivity:

In MS Windows regular file names and folder names are case-insensitive: e.g. You cannot place 2 files called "MYFile" and "myfILE" in the same folder. In Linux/Unix names are case-sensitive and 2 such files can co-exist. Jeeva tests on OS-type and takes care of this automatically when comparing and copying. However 2 command line options: --cs and --ci permit user to over-ride and force case-sensitivity/case-insensitivity. If there is another OS out there which is also case-insensitive then use --ci but normally it should not be necessary to use them.

Customization

There is not much that can be done in the way of customization. Whatever there is is done through editing the file jeeva.py, therefore it is not available to Windows precompiled binary users. Near the top of the file are the folowing lines which change formatting of output for the --overlap option: 

#used for formatting of output for --overlap
INDENT_INCREMENT = "   "
INDENT_ARROW = "|->"
WIDTH_PRINTINDENTWIDTH = 65      #change this if want to narrow/widen output field for the --overlap output

As mentioned earlier, if your source and and destination roots are always the same you can hardwire them into your jeeva.bat file, (Apple/Linux: into a shell script or alias command) and not have to worry about typing the whole thing every time.
